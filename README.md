# Your Med Bot — Telegram-бот записи на приём (АСМС)

Telegram-бот для записи к врачу: первичный и вторичный приём, умный анамнез, выбор слотов, напоминание за 24 часа. Все времена — **по Москве**. Токен бота хранится только в `.env` (в репозиторий не попадает).

---

## Обзор сценариев

- **Старт:** если у пользователя уже есть записи в БД — выбор «Первичный» / «Вторичный» (вторичный = визиты к тому же врачу за последние 6 мес). Если записей нет — сразу первичный приём.
- **Вторичный приём:** выбор врача из списка (к кому обращался за 6 мес) → слоты по 10 мин (приоритет — окна за 3 дня; если меньше 10 — показываются ближайшие 10).
- **Первичный, знает врача:** выбор врача из списка → предложение умного анамнеза. Без анамнеза — слоты по 20 мин (3 дня или 10 окон). С анамнезом — описание симптомов → 10–20 вопросов по одному сообщению, черновик сохраняется, при возврате — «Продолжить?» → сводка, «Всё верно» / исправление по номеру вопроса → слоты по 10 мин. При записи в БД сохраняется текст анамнеза.
- **Первичный, не знает врача:** описание симптомов → подбор до 3 врачей (заглушка) → выбор врача → тот же блок умного анамнеза и слотов.
- **После записи:** подтверждение пользователю; напоминание за 24 ч с кнопкой «Отменить запись».

Подбор врачей по симптомам и список вопросов анамнеза реализованы **заглушками** (можно заменить на свой API или модуль).

## Структура проекта (бот)

| Путь | Назначение |
|------|------------|
| `bot/` | Код бота: FSM, обработчики, клавиатуры, заглушки алгоритмов |
| `db/` | SQLite: схема, CRUD (пользователи, врачи, слоты, записи, черновики анамнеза) |
| `scripts/seed_db.py` | Генерация тестовых данных: врачи, слоты на 7–14 дней |
| `config.py` | Токен из окружения, часовой пояс Москва |
| `entrypoint.py` | При первом запуске в Docker заполняет БД, затем запускает бота |

## Окружение для бота

Рекомендуется использовать **отдельное виртуальное окружение** только для бота, чтобы не смешивать зависимости с другими задачами (например, с экстрактором Википедии).

- Папка окружения: `.venv` в корне проекта (уже в `.gitignore`).
- Все команды ниже для **локального** запуска выполняйте из корня проекта, **после активации** окружения.

---

## Запуск локально (Python + venv)

### Требования

- Python 3.11 или 3.12
- Токен бота от [@BotFather](https://t.me/BotFather)

### Пошаговая инструкция

1. **Перейдите в корень проекта**
   ```bash
   cd /путь/к/your-med-bot
   ```

2. **Создайте виртуальное окружение**
   ```bash
   python3 -m venv .venv
   ```

3. **Активируйте окружение**
   - macOS / Linux:
     ```bash
     source .venv/bin/activate
     ```
   - Windows (cmd):
     ```bash
     .venv\Scripts\activate.bat
     ```
   - Windows (PowerShell):
     ```bash
     .venv\Scripts\Activate.ps1
     ```
   В активированном окружении в начале строки терминала будет `(.venv)`.

4. **Установите зависимости бота**
   ```bash
   pip install -r requirements.txt
   ```

5. **Настройте токен**
   ```bash
   cp .env.example .env
   ```
   Откройте `.env` и укажите:
   ```
   BOT_TOKEN=ваш_токен_от_BotFather
   ```

6. **Инициализируйте БД и тестовые данные**
   ```bash
   python -m scripts.seed_db
   ```
   Опционально добавить тестовые записи (для проверки вторичного приёма):
   ```bash
   python -m scripts.seed_db --with-appointments
   ```

7. **Запустите бота**
   ```bash
   python -m bot.main
   ```
   Остановка: `Ctrl+C`.

   Если после обновления кода слотов нет — заново заполните БД: `python -m scripts.seed_db`.

## Запуск через Docker

Удобно запускать с любых устройств без установки Python и зависимостей.

### Требования

- [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/)

### Пошаговая инструкция

1. **Перейдите в корень проекта**
   ```bash
   cd /путь/к/your-med-bot
   ```

2. **Настройте токен**
   ```bash
   cp .env.example .env
   ```
   В файле `.env` укажите:
   ```
   BOT_TOKEN=ваш_токен_от_BotFather
   ```

3. **Соберите образ и запустите контейнер**
   ```bash
   docker compose up -d --build
   ```
   При первом запуске автоматически создаются таблицы и заполняются тестовые данные (врачи, слоты на 14 дней). БД хранится в Docker-томе `botdata` и сохраняется между перезапусками.

4. **Проверьте, что бот работает**
   ```bash
   docker compose logs -f bot
   ```
   В логах не должно быть ошибок; в Telegram отправьте боту команду `/start`.

5. **Остановка**
   ```bash
   docker compose down
   ```

### Дополнительные команды (Docker)

- **Добавить тестовые записи** (для проверки вторичного приёма):
  ```bash
  docker compose run --rm bot python -m scripts.seed_db --with-appointments
  ```
- **Пересобрать и запустить** (после изменения кода):
  ```bash
  docker compose up -d --build
  ```
- **Сбросить данные** (удалить том с БД и заново заполнить при следующем запуске):
  ```bash
  docker compose down -v
  docker compose up -d --build
  ```

# Экстрактор медицинских данных (Vibe Hack)

Автоматизированный инструмент для извлечения медицинского анамнеза, симптомов и клинических описаний из русскоязычной Википедии для создания структурированной базы данных для обучения медицинских LLM.

## Обзор экстрактора

Проект реализует пайплайн для:
1. Чтения списка медицинских заболеваний из Wikidata/Wikipedia (`query.json`).
2. Скрапинга релевантных разделов (Симптомы, Клиническая картина, Жалобы и др.) с использованием API Википедии.
3. Генерации структурированной базы данных в формате JSON (`disease_anamnesis_database.json`), подходящей для дообучения нейросетей или создания медицинских RAG-систем.

## Структура (экстрактор)

- `data.py` — логика взаимодействия с API и обработки текста.
- `query.json` — список заболеваний и метаданные из Wikidata.
- `disease_anamnesis_database.json` — выходной файл с извлечёнными данными.

## Скачивание данных

[Ссылка для экстракции json из вики](https://query.wikidata.org/#SELECT%20%3Fitem%20%3FitemLabel%20%3FarticleTitle%20WHERE%20%7B%0A%20%20%3Fitem%20wdt%3AP31%2Fwdt%3AP279%2a%20wd%3AQ12136.%20%23%20Instance%20of%20%28or%20subclass%20of%29%20%22disease%22%0A%20%20%3Farticle%20schema%3Aabout%20%3Fitem.%0A%20%20%3Farticle%20schema%3AisPartOf%20%3Chttps%3A%2F%2Fru.wikipedia.org%2F%3E.%0A%20%20%3Farticle%20schema%3Aname%20%3FarticleTitle.%0A%20%20SERVICE%20wikibase%3Alabel%20%7B%20bd%3AserviceParam%20wikibase%3Alanguage%20%22ru%22.%20%7D%0A%7D)

## Установка и запуск экстрактора

Рекомендуется **mamba** или **conda**:

```bash
mamba install -c conda-forge wikipedia-api pandas tqdm
```

В `data.py` укажите User Agent. Затем:

```bash
python data.py
```

## Схема выходных данных

```json
[
    {
        "Summary": "Краткое описание заболевания...",
        "Симптомы": "Подробное описание симптомов...",
        "Disease_Name": "Название болезни",
        "Wikipedia_Title": "Заголовок статьи",
        "Wikidata_URL": "http://www.wikidata.org/entity/Q..."
    }
]
```

## Примечание

Экстрактор ориентирован на разделы, связанные с **анамнезом** и **симптоматикой**. Разделы с названиями вне списка `TARGET_SECTIONS` в `data.py` пропускаются.
