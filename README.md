# Your Med Bot

Telegram-бот для умного сбора анамнеза и записи пациента к врачу.  
Цель проекта — сократить время первичного опроса: бот последовательно уточняет жалобы, выделяет симптомы, оценивает срочность, рекомендует профиль специалиста и переводит пользователя в запись на приём.

## Зачем это нужно

В реальном потоке пациент часто приходит с неструктурированным описанием состояния.  
Бот превращает свободный текст в понятную для врача карточку и помогает быстрее маршрутизировать пациента:

- снижает нагрузку на первичную коммуникацию;
- ускоряет запись к релевантному специалисту;
- сохраняет историю симптомов в одном диалоге;
- добавляет напоминание о визите и возможность отмены записи.

## Что делает бот

- ведёт сценарий первичного и вторичного приёма;
- проводит AI-опрос для сбора анамнеза;
- извлекает симптомы из диалога;
- оценивает срочность и предлагает профиль врача;
- подбирает доступные слоты и оформляет запись;
- отправляет напоминание за 24 часа с кнопкой отмены.

## За счёт чего работает

- `aiogram` + FSM для диалогового сценария в Telegram;
- LLM-агент в `llm/agent.py` для интеллектуального опроса пациента;
- RAG-поиск по медицинской базе на основе `FAISS`;
- SQLite-слой (`db/`) для хранения пользователей, врачей, слотов, записей и черновиков.

## Ключевой пользовательский поток

1. Пациент описывает жалобы в свободной форме.
2. AI-агент задаёт уточняющие вопросы и накапливает медицински значимый контекст.
3. После достижения достаточной полноты бот формирует рекомендации по врачу.
4. Пациент выбирает врача и временной слот.
5. Бот сохраняет запись и отправляет напоминание перед приёмом.

## Кратко про данные и экстрактор

Для подготовки медицинской базы использовался отдельный экстрактор данных из русскоязычной Википедии (скрипты `data.py`, `clean_db.py`, `ingest.py` и их версии в `scripts/`).  
Результаты подготовки (датасеты, очищенные JSON, FAISS-индекс, ноутбуки) оставлены в репозитории как артефакты исследовательского этапа и воспроизводимости пайплайна.

## Структура репозитория

- `bot/` — Telegram-бот и сценарии записи;
- `llm/` — логика AI-агента анамнеза;
- `db/` — схема и CRUD для базы данных;
- `doctor/` — датасет и утилиты рекомендаций по симптомам;
- `data/` — подготовленные медицинские данные и индекс для RAG;
- `scripts/` — вспомогательные скрипты подготовки данных и генерации тестовой БД;
- `tests/` — текущие тестовые/проверочные сценарии проекта.

## Ограничения

Проект не является медицинским диагностическим сервисом и не заменяет очную консультацию врача.  
Результаты AI-агента используются как предварительный сбор анамнеза для маршрутизации и ускорения записи.
